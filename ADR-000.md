FreshRSS 插件开发说明

项目名称

freshRSS-QualityFilter

⸻

项目目标

开发一个适用于 FreshRSS 1.29.x 的扩展，在 RSS 文章导入数据库之前，对文章进行质量过滤（Quality Filter）。

要求：

* 不修改 FreshRSS Core。
* 使用官方 Extension API。
* 使用 EntryBeforeInsert Hook，在写入数据库之前完成过滤。
* 不符合条件的文章直接跳过导入（Skip Import）。
* 所有配置均可在 FreshRSS 后台配置。
* 完全兼容 Docker 部署。

⸻

技术要求

语言：

* PHP 8.2+

兼容版本：

* FreshRSS 1.29.x

代码规范：

* PSR-12
* 尽量使用 readonly
* 使用 strict_types
* 全部 UTF-8

不要：

* 修改 FreshRSS Core
* 修改数据库结构
* Monkey Patch
* 使用 Composer 外部依赖

FilterResult：

class FilterResult
{
    public bool $passed;
    public string $reason;
    public ?string $matchedRule;
    public int $score;

继续

开发要求

整个项目保持高内聚、低耦合。

遵循 SOLID 原则。

所有过滤逻辑必须可扩展，不允许将所有逻辑写在 extension.php 中。

未来新增一个过滤器，应仅需新增一个实现 FilterInterface 的类，并注册到 FilterPipeline，而无需修改已有过滤器代码。
